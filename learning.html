Set Up a Loyalty Program

Section Details
Loyalty Program Setup
    Set up a loyalty program in the Loyalty Management app You can create and activate the program, which supports both B2C(customers) and B2B (partners) models.
Tier Groups and Tiers
    Create a tier group and associate it with the loyalty program. Define three key attributes: Tier Period, Qualifying Period, and Tier Model (Fixed or Anniversary). Create tiers based on points, offering different rewards per tier.
Program Currencies
    Define Qualifying Points (used for tier progression) and Non-Qualifying Points (redeemable for rewards). Set up expiration rules, like theFixed Model or Activity Model, for non-qualifying points.
Escrow Points
    Use escrow to temporarily hold points after a transaction before crediting them to members, ensuring proper handling in case of returns.
Partners and Partner Products
    Add partners and associate their products with the program. Partners can be accrual partners (members earn points) or redemption partners (members spend points). Products can be assigned individually or by category.
Program Members
    Enroll members by inviting them to sign up. Restrict membership to individuals or allow group membership, where members can contribute points to a shared balance.
Next Step
    Once the loyalty program, tier groups, currencies, partners, and members are set up, the next step is to focus on defining member rewards.


Key Features and Benefits of Loyalty Management and Promotions

Feature Details
Benefits
    Benefits reward loyalty members to increase their program engagement. As members progress to higher tiers, they receive more benefits.
Benefit Type
    Groups benefits (e.g., Priority Phone Support, Standard Web Support) created under each type.
Benefit Action
    Automatically triggers actions, such as creating a Service Entitlement when a support benefit is defined.
Promotions
    Engage loyalty members with valuable offers tied to specific times or sales cycles. They increase product sales and business revenue, offering vouchers or loyalty points.
Standard Promotions
    Provide immediate benefits to members based on single transactions (e.g., 20% off jackets on a specific date).
Cumulative Promotions
    Reward members for long-term behavior, such as spending a certain amount over time (e.g., 1000 points for $1000 spent in January).
Joint Promotions
    Promotions involving partners with shared contribution and liability (e.g., 40% and 60% share between two brands).
Global Promotions Management
    Design pricing and non-pricing promotions for loyalty programs or as part of the company’s marketing strategy. Run granular promotions with in-built limits.
Quick Promotions
    Create a promotion in four steps using a Quick Promotion Wizard.
Promotion Creation
    Define promotion types, program association, enrollment requirements, start/end dates, and reward criteria (e.g., vouchers or points)
Add Partner Products
    Associate specific products or categories with a particular promotion.
Set Up Promotions with Data Cloud
    Use Data Cloud to create customer segments and launch promotional campaigns
Set Up Promotions with Campaigns
    Use Salesforce Reports for customer segments when Data Cloud isn’t available, and run promotional campaigns.
Actionable Segmentation
    Find loyalty members eligible for promotions using criteria defined by program managers, creating segmentation lists
Apply Promotions
    Use Flow actions to apply promotions to members and update their progress in real-time.
Promotion Calendar
    Visualize promotions on a calendar to check events and find opportunities for new promotions.
Clone Promotions
    Clone existing promotions for seasonal or repetitive use, edit if necessary, and activate the new promotion.
SView & Track Member Promotions
    Track members’ progress for Cumulative promotions, view promotions offered to members.

Create Classes and Objects

Topic Summary
Object-Oriented Definition
    In programming, object-oriented means that the code focuses on describing objects, which have unique characteristics. Examples include people, accounts, or flowers.
Classes
    A class is a blueprint for objects, defining shared characteristics (variables) and behaviors (methods) of all objects of that class. For example, a flower class might include variables like color and height, and methods like grow and wilt.
Methods
    Methods represent behaviors or actions for objects of a class. For example, a flower class might have methods such as grow, pollinate, or wilt.
Parameters
    Parameters are variables that serve as placeholders in methods, allowing different values to be passed into the method. For example, a parameter like numberOfPetals in the wilt method receives a value to operate on.
Return Types
    Methods may return a value or not. The return type specifies what kind of value the method will return (e.g., integer, string) or can be void, meaning it returns nothing. For example, the wilt method may return an integer representing the number of petals left after wilting.
Objects
    Objects are instances of a class, inheriting its variables and methods. For example, a tulip object is an instance of the flower class, inheriting variables like color and height and methods like grow and wilt.
Object Creation
    To create an object, you instantiate a class. For example, Flower tulip = new Flower(); creates a tulip object from the Flower class. Dot notation is used to call methods on objects (e.g., tulip.grow();).

                        public String make, model;

                            this.make = make;

Introduction to sObjects and DML in Salesforce Apex

In Salesforce development, sObjects and Data Manipulation Language (DML) are crucial for managing and interacting with data. Here's a quick overview to get you started:

Key Concepts


  1. What is an sObject?

    An sObject is an Apex data type that represents a Salesforce object, like an Account, Contact, or Opportunity.
    Unlike most data types that hold single values, sObjects are containers that can hold multiple fields and values related to a Salesforce record. For example, an Account sObject can store attributes like Name, Phone, and AccountNumber.


  1. DML Basics

    Data Manipulation Language (DML) in Apex lets you insert, update, and delete records in Salesforce through code, similar to how you might use tools like Data Loader for bulk uploads.
    DML commands include Insert, Update, and Delete to perform actions directly on sObject records.


Creating and Managing Records with sObjects and DML


Setting Field Values: Use dot notation to assign values to sObject fields, e.g., Account myAcct = new Account(); myAcct.Name = 'The Tea Factory';.
Inserting Records: To add records to the Salesforce database, declare a new sObject and use the Insert DML command:


            public class NewAccounts {
                public static void sObjectsInsert(){
                    Account store = new Account();
                    store.Name = 'The Tea Factory';
                    insert store;
                }
            }

Bulk Record Insertion with Loops


Using Lists and Loops: To add multiple records, leverage lists and a loop structure :
 

            public class NewAccounts {
                public static void sObjectsInsert(Integer count){
                    List teaFactoryAccounts = new List();
                    for(Integer i = 1; i <= count; i++){
                        Account store = new Account(Name = 'The Tea Factory ' + i, AccountNumber = '35629' + i);
                        teaFactoryAccounts.add(store);
                    }
                    insert teaFactoryAccounts;
                }
            }

Running this method with count = 3 creates three Account records.


Best Practice: Bulkification


Why Bulkify?

Inserting multiple records at once is efficient and crucial for scalability.
Instead of inserting records individually, adding them as a batch helps stay within Salesforce's governor limits, which cap insert operations per transaction.
Bulkification helps your application scale by performing mass insertions within limited resources.


Conclusion

This introduction to sObjects and DML in Salesforce sets a foundation for working with Salesforce data through code, making your applications scalable and efficient.

Define Sets and Maps

Collection Type Description Examples When to Use
List
    An ordered collection of items, each with a specific index. Duplicates are allowed. 
List teaTypes = new List{'Black', 'White', 'Herbal'};
    Use when order matters and duplicates are allowed.
Set
    An unordered collection of unique items. No duplicates allowed.
    Set teaTypes = new Set{'Black', 'White', 'Herbal'};
    Use when duplicates are not allowed and order doesn't matter.
Map
    A collection of key-value pairs. Each key must be unique, but values can be duplicated. 
Map teaTypes = new Map();
                        teaTypes.put('Black', 'Earthy');
    Use when you need to map a unique key to a value (e.g., country code to country name).

Key Differences:


List: Ordered, allows duplicates.
Set: Unordered, no duplicates.
Map: Key-value pairs, keys are unique, values can repeat.

Methods:


List: Add, remove, get by index.
Set: Add, check existence, no indexing.
Map: put(key, value), get(key) to retrieve values.

Use Cases:

Use a List: when you need to preserve the order of items.
Use a Set: when you need unique items and don’t care about the order.
Use a Map: when you need to associate keys with specific values (e.g., storing a person's name with their ID).


Use a For Loop to Iterate Through a List

Type of For Loop Description Syntax Example When to Use
Traditional For Loop Iterates a specific number of times, with a counter variable initialized, incremented, and checked. for(Integer i = 0; i < 5; i++) { System.debug('The number is ' + i); } Use when you know how many times the loop should run, or the loop depends on a counter variable.
List/Set Iteration For Loop Iterates through all elements in a list or set without needing a counter or condition. for(String t : teaList) { System.debug('We have ' + t); } Use to traverse lists or sets, automatically iterating through all elements.
