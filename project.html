Tennis Club App

About the Tennis Club App

The road to developing the Asca Tennis Club app was defined by my experiences as a club member.
I regularly faced issues with court reservations, where coaches and players would end up using courts I had reserved.
Communication with other members was also challenging, as there was no platform to exchange contact details or set up matches.

With my Salesforce knowledge and experience, I saw an opportunity to solve these problems.
The app was developed to streamline court reservations, foster communication among members, and offer a more organized tennis club experience for everyone.

App's Requirements

  • Purpose: To solve issues with court reservations and improve communication among members.
  • Target Audience: Club members, coaches, and administrators who manage club logistics.
  • Key Features: Court Reservations, Member List & Communication, Coach Directory, Chatter Feed, and Gallery.

User Experience

The app offers a user-friendly interface for booking courts and connecting with other members.
The Chatter feed enhances community interaction, while admin features allow for easy management of members, reservations, and coaches.

App Design & Development

Design Tools: Lightning App Builder for designing the user interface.

Backend Development: Apex Classes, Lightning Web Components (LWC), Custom Objects, and Salesforce Data Management using SOQL/SOSL.

list of tools used to develop the Tennis Club app

  1. Object Manager - For creating and managing custom objects like Club_Members__c, Coaches__c, Court_Reservation__c, and Tournaments__c.
  2. Lightning Web Components (LWC) - To build custom user interfaces, such as the club members list, court reservation system, and tournament displays
  3. Flow Builder - To automate processes, like managing the gallery feature and streamlining member interactions.
  4. Component Visibility - To control which components are visible to specific users based on their profiles.
  5. Profiles and Permission Sets - To manage access levels for different user roles, such as Admin and Players
  6. Chatter - For adding a communication feed, improving member interactions.
  7. Page Layouts - Customizing how records are displayed for different users.
  8. App Builder - For creating the overall layout and navigation of the app.
  9. Apex - To handle specific backend logic, such as fetching lists of users or automating advanced processes
  10. JavaScript - For adding interactivity and dynamic features within the LWC components.
  11. HTML - To structure the content and layout of the app interfaces.
  12. Visual Studio - Used as the primary code editor for developing the Lightning Web Components and custom code.
